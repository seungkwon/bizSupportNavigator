  - 채팅 페이지 UX 개선: 네비바의 정책 미지정("채팅 상담") 진입점 삭제, 대시보드 매칭 카드에 체크박스를 추가해 체크 시 해당 정책으로 targeted 채팅 바로 이동. 채팅 결과 화면에서 방금 답변한 대상 정책을 근거(판정 사유)와 함께 상단에 강조 표시하고, 나머지는 "다른 추천 정책"으로 분리 — 클릭하면 그 정책으로 타겟 전환. `chat_service.py`의 `_serialize_cached`(세션 재연결 시 즉시 전송되는 재개 메시지)가 `match_results`에 없는 `title`을 누락해 다른 추천 정책 제목이 빈 칸으로 보이던 버그 수정.
  - 채팅 질문 프로토콜을 1문항씩 왕복(`question`/`answer`)에서 일괄 질문/일괄 답변(`question_batch`/`answer_batch`)으로 변경 — 대상 정책의 미확인 요건을 한 번에 모두 보여주고 전부 답한 뒤에만 재계산. 첫 라운드는 대시보드의 직전 재계산 결과(`match_results.reasons`)를 그대로 재사용해 질문을 즉시 표시(LLM 호출 없이, company_facts 대비 임베딩 유사도로 이미 해결된 항목은 로컬에서 걸러냄), 이후 라운드만 실제 `judge_policy` 호출. 확인 대상에 정보부족뿐 아니라 미충족(LLM 추론, 사용자 미확인)도 포함해 재확인 질문. 최종 재계산 시에도 캐시된 점수에 미해결 항목이 없는 후보는 재판정을 건너뛰고 캐시 재사용 — 변동 가능성이 없는 후보에 대한 불필요한 OpenAI 호출 제거.
  - 사용자가 직접 "아니오"라고 답해 확정된 미충족 판정을 LLM이 그냥 추론한 미충족과 구분 — `MatchReason.confirmed` 필드 추가, 프론트엔드에서 "미충족 확정"(빨강)과 일반 미충족(주황)을 다른 배지 색으로 표시.
  - 확정된 미충족 요건이 하나라도 있으면(신청 자체가 불가능한 정책이므로) 점수를 0으로 고정. 그런 정책은 재계산 대상에서도 제외(캐시 그대로 재사용, OpenAI 호출 안 함) — 단 `company_facts` 관리 화면에서 그 근거가 된 답변을 삭제/수정하면 확정이 풀리면서 다시 재계산 대상에 포함됨. 실제 fact 삭제 → 재계산 → 점수 복원까지 end-to-end로 검증 완료.
  - "예"라고 답한 요건인데도 LLM이 (인구통계 등 더 확실한 근거로) 미충족으로 판정하는 모순 사례를 일반 미충족과 구분 — `MatchReason.conflicting` 필드 추가, "미충족 (답변과 다름)"으로 표시(파란 배지). `confirmed`/`conflicting`은 매칭된 `company_facts` 답변이 각각 "아니오"/"예"인지로 구분(`best_matching_answer`, 임베딩 유사도, LLM 호출 없음). 테스트 스크립트가 만든 demo-001/demo-002의 테스트용 `company_facts`/`match_results`/채팅 세션은 정리함.
  - 채팅에서 다음 라운드 질문 배치가 오면 이전 라운드 질문(+답변)이 화면에서 사라지던 것을 수정 — 답변 완료된 질문은 회색 카드로 위에 계속 쌓아 보여주고, 새 라운드 질문은 그 아래에 이어서 표시(예: 2개 답변 후 1개 더 나오면 총 3개가 누적 표시).
